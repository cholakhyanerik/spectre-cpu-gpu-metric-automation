# 📊 Spectre CPU/GPU Metric Automation

Инструмент для автоматизированного мониторинга производительности десктопного приложения **Spectre Terminal**. 

Скрипт работает в режиме «ассистента-наблюдателя» (Watcher): он запускает указанный билд приложения, незаметно собирает метрики (ЦП, ОЗУ, ГП) в фоновом режиме, пока QA-инженер проводит ручное (мануальное) тестирование. Инструмент автоматически рекурсивно отслеживает все дочерние процессы (модальные окна) и формирует детальные отчеты с графиками сравнения.

## 🎯 Цель проекта
В соответствии с регламентом тестирования Spectre Terminal: **Любая деградация показателей ЦП или ГП по сравнению с веткой main является абсолютным блокером.** Этот проект автоматизирует сбор доказательной базы (JSON-отчеты и PNG-графики) для перевода задач в статус `Bugs Found`.

## 🛠 Требования
* Python >= 3.14
* Пакетный менеджер [uv](https://github.com/astral-sh/uv)

## 🚀 Установка и настройка

1. **Клонируйте репозиторий и перейдите в папку проекта:**
   ```bash
   git clone <your-repo-url>
   cd spectre-cpu-gpu-metric-automation

```

2. **Установите зависимости через `uv`:**
```bash
uv sync
uv add pytest-html python-dotenv GPUtil matplotlib setuptools

```


3. **Создайте файл конфигурации:**
В корне проекта создайте файл `.env` (он добавлен в `.gitignore` и не попадет в репозиторий). Скопируйте туда следующие настройки:
```env
# Пути к билдам (DEV - эталон, FUTURE - тестируемая фича)
# Пример Windows: C:\path\to\spectre-terminal.exe
# Пример macOS: /Applications/Spectre Terminal.app/Contents/MacOS/Spectre Terminal
DEV_BUILD_PATH=
FUTURE_BUILD_PATH=

# Допустимая деградация (Блокеры)
CPU_TOLERANCE_PERCENT=0.1
RAM_TOLERANCE_MB=1.0
GPU_TOLERANCE_PERCENT=0.1
```



## 🔄 Рабочий процесс (QA Workflow)

Процесс теперь выполняется за один запуск, чтобы минимизировать погрешности:

### Шаг 1: Подготовка
1. Скачайте стабильный билд (DEV) и тестируемый билд (FUTURE).
2. В файле `.env` укажите пути к обоим билдам (`DEV_BUILD_PATH` и `FUTURE_BUILD_PATH`).

### Шаг 2: Тестирование
1. Запустите тест:
```bash
uv run pytest src/tests/test_application.py -s --html=report.html --self-contained-html -W ignore
```

2. Скрипт одновременно откроет обе версии приложения.
3. Проведите ручное тестирование сценариев параллельно в обоих окнах (DEV и FUTURE).
4. Закройте оба приложения. Скрипт сохранит метрики и сгенерирует графики для обоих, а также автоматически их сравнит.

## 📊 Результаты и Отчеты

Все результаты тестирования автоматически сохраняются в директорию `reports/`:

* `reports/latest/` — актуальные данные последнего прогона:
* `dev_baseline_metrics.json` — сохраненный эталон.
* `dev_metrics_chart.png` — график потребления стабильной ветки.
* `feature_metrics_chart.png` — график потребления тестируемой фичи.
* **`comparison_chart.png`** — наглядный график наложения (Dev vs Feature) для прикрепления в Jira.


* `reports/history/` — исторический архив всех запусков (формат: `run_YYYYMMDD_HHMMSS.json`) для ретроспективного анализа.
* В корне проекта генерируется `report.html` с логами pytest.

Если потребление ресурсов превышает допустимую норму (`TOLERANCE` из `.env`), тест завершится со статусом **FAILED**, что является сигналом для возврата задачи разработчику.
